pwd : la commande pwd permet de savoir dans quel répertoire vous vous trouvez.

ls: La commande ls répertorie les fichiers et les répertoires 
    dans le système de fichiers et affiche des informations détaillées sur eux.
    
cd : (change directory) Permet de changer de répertoire

tab : Auto-completer

mkdir : (make directories) permet de créer des répertoires.

touch : Le but premier de cette commande est de mettre à jour le “timestamp” d'un fichier, 
        c'est-à-dire la date de dernière modification du fichier.

git status : La commande git status affiche la liste des fichiers modifiés ainsi que les fichiers
             qui doivent encore être ajoutés ou validés.

git add : Cette commande met à jour l'index en utilisant le contenu actuel trouvé dans l'arbre de travail, 
          pour préparer le contenu mis en scène pour le prochain commit.

git commit -m : La commande git commit permet de valider les modifications apportées au HEAD. 
                Notez que tout commit ne se fera pas dans le dépôt distant.

git pull : Pour fusionner toutes les modifications présentes sur le dépôt distant dans le répertoire de travail local,
           la commande pull est utilisée.
git push : Git push est une autre commandes GIT de base. Un simple push envoie les modifications locales 
           apportées à la branche principale associée :
git log : L’ exécution de cette commande génère le log d’une branche. Un exemple de sortie.

git tag : Le marquage est utilisé pour marquer des commits spécifiques avec des poignées simples.

git clone : La commande git clone est utilisée pour la vérification des dépôts. Si le dépôt se trouve
            sur un serveur distant.
